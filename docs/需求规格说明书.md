## 1. 项目概述

### 1.1 项目背景

情侣相处过程中，极易出现情绪低落、生气、闹矛盾等场景。此时缺乏一套**专属、温馨、即时可用**的情绪安抚与互动渠道，容易导致矛盾激化、冷战升级。市面上的情侣 App 需要下载、注册、学习成本高，且内容为通用模板，缺乏专属感。

为解决上述问题，开发一套**专属情侣亲密度升温 Web 系统**——无需下载 App，手机浏览器扫码或点链接即用，主打五大核心能力：

| 核心能力 | 说明 |
|----------|------|
| **情绪治愈** | 生气/难过/委屈场景下，一键触发专属哄宠模式 |
| **专属偏爱** | 所有话术、福利、承诺 100% 由男主自定义，杜绝通用套话 |
| **互动升温** | 每日打卡 / 真心话 / 积分商城 / 悄悄话，常态化增进感情 |
| **哄宠专属** | 认错保证书、豁免卡、睡前故事，专属宠爱拉满 |
| **私密安全** | 双人密码访问，数据私有存储，守护甜蜜回忆 |

### 1.2 项目目标

| 目标维度 | 具体指标 |
|----------|----------|
| **轻量化部署** | SpringBoot 单 jar 包启动，云服务器 / 本地服务器均可部署 |
| **便捷访问** | 手机扫码或点击 URL 直达，零安装、零注册、零学习成本 |
| **核心情绪解决** | 生气 / 难过 / 委屈 / 低落四大场景全覆盖，快速平复负面情绪 |
| **常态化升温** | 每日互动 + 甜蜜记录 + 积分福利，持续提升亲密度 |
| **专属私密属性** | 仅情侣二人可用，所有数据专属二人，严格隐私保护 |

### 1.3 运行环境

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **服务端** | Java 8 + SpringBoot 2.6.x | 轻量化，单 jar 部署，支持 7×24 常驻运行 |
| **数据库** | MySQL 5.7 / 8.0 | 轻量级，存储全部业务数据 |
| **持久层** | MyBatis-Plus 3.x | 简化 CRUD，减少样板代码 |
| **前端模板** | Thymeleaf + Vue 3 (CDN) | 移动端自适应，按需融合 |
| **客户端** | 任意手机浏览器 | 微信 / 抖音 / Safari / Chrome / 系统浏览器 |
| **访问方式** | 固定 URL / 二维码扫码 | 内网穿透（如 frp）或云服务器公网 IP |

## 2. 用户角色说明

本系统为**双人专属私密系统**，无复杂权限体系，仅包含两类角色：

| 角色 | 身份 | 核心职责 | 权限范围 |
|------|------|----------|----------|
| **男主** | 开发者 / 管理员 | 后台录入情话、福利、道歉话术、承诺书、故事；查看情绪记录与兑换记录 | 后台登录 → 话术管理 / 福利管理 / 纪念日管理 / 数据查看 / 素材管理 / 系统设置 |
| **女主** | 核心使用用户（女友） | 日常使用所有前端治愈、互动功能；情绪不佳时为核心使用对象 | 前端页面 → 情绪急救 / 互动升温 / 恋爱记录 / 偏爱特权 |

> **设计约束**：前端所有页面无需登录（仅凭访问密码进入）；后端管理台需男主账号密码登录。

## 3. 核心功能需求（前端—女友端）

> **设计原则**：所有功能手机端自适应，界面温柔甜美（粉白淡紫主色调），操作极简，情绪不佳时一键点开即用。

| 字段 | 说明 |
|------|------|
| **积分获取途径** | 每日打卡 +10 / 情绪平复 +5 / 完成真心话 +5 / 完成默契测试 +8 / 每日首次访问 +2 |

## 4. 后端管理功能需求（男主端）

> **设计原则**：后台简洁轻量化，仅男主可登录，所有操作 2 步内完成。

### 4.1 登录与安全

| 功能 | 说明 |
|------|------|
| **登录方式** | 账号 + 密码登录，密码 MD5 + Salt 加密存储 |
| **Session 管理** | 登录 Token（JWT），有效期 24 小时，超时自动退出 |
| **密码修改** | 后台可修改登录密码（需验证旧密码） |

## 9. 非功能性需求

### 9.1 界面 UI 需求

| 需求项 | 详细规格 |
|--------|----------|
| **整体风格** | 温柔治愈、甜美简约，拒绝花哨 |
| **主色调** | 粉色 `#FFB6C1`、浅白 `#FFF5F7`、淡紫 `#D5C6E0`，参考色板见附录 |
| **字体** | 系统默认中文字体（苹方/PingFang SC），标题可使用圆体，字号≥14px |
| **按钮** | 最小触控区域 44×44px（符合 iOS HIG），圆角 20px+ |
| **动画** | 弹窗缩放 `cubic-bezier(0.34, 1.56, 0.64, 1)`，爱心粒子 CSS @keyframes，过渡 `0.3s ease` |
| **响应式** | `viewport: width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no` |
| **音效** | 默认关闭背景音乐，提供开关按钮，按钮位于页面右上角 |

### 9.2 性能需求

| 指标 | 目标值 | 验证方式 |
|------|--------|----------|
| **首屏加载** | ≤ 1 秒（4G 网络） | Chrome DevTools Lighthouse |
| **API 响应** | ≤ 200ms（P95） | 服务端日志统计 |
| **图片加载** | 缩略图 ≤ 50KB，原图 ≤ 500KB | 前端上传时自动压缩 |
| **可用性** | 7×24 小时稳定运行 | 进程守护（systemd / supervisor） |
| **并发** | 支持 2 人同时使用无卡顿（用户量小，无需高并发设计） | — |

### 9.3 安全需求

| 需求项 | 实现方案 |
|--------|----------|
| **前端访问密码** | 数据库中存储密码 Hash（MD5+Salt），前端输入后 POST 验证，通过后设置 Session Cookie（24h 有效） |
| **后台登录** | 账号+密码 → JWT Token，接口拦截验证，Token 过期需重新登录 |
| **SQL 注入防护** | MyBatis 参数化查询（`#{}`），禁止 `${}` 拼接用户输入 |
| **XSS 防护** | Thymeleaf 默认转义 + 前端输出过滤 |
| **文件上传安全** | 限制文件类型白名单（jpg/png/gif/webp/mp3/m4a），限制大小（图片 10MB，音频 20MB），后端校验 MIME 类型 |
| **隐私保护** | 所有数据不上传公开服务器，API 禁止未授权访问，上传文件目录禁止目录遍历 |

### 9.4 兼容性需求

| 平台 | 浏览器 | 最低版本 |
|------|--------|----------|
| **iOS** | Safari | iOS 13+ |
| **iOS** | 微信内置浏览器 | 微信 8.0+ |
| **Android** | Chrome | Android 8.0+ |
| **Android** | 微信内置浏览器 | 微信 8.0+ |
| **Android** | 系统浏览器（小米/华为/OPPO/vivo） | Android 8.0+ |
| **其他** | 抖音内置浏览器 | 最新版 |

> **不兼容范围**：平板端、桌面端不做适配（可访问但不保证体验）。

### 9.5 数据可靠性

| 需求项 | 实现方案 |
|--------|----------|
| **数据备份** | ① 数据库每日自动备份（cron + mysqldump） ② 上传文件目录定期 rsync 备份 |
| **数据恢复** | 保留最近 30 天备份，支持手动恢复 |

## 附录 C：浏览器 TTS 语音方案

```javascript
// 前端语音朗读核心代码示意
function speak(text, rate = 0.85) {
  if (!window.speechSynthesis) {
    console.warn('浏览器不支持语音合成');
    return;
  }
  const utterance = new SpeechSynthesisUtterance(text);
  utterance.lang = 'zh-CN';
  utterance.rate = rate;      // 0.5-1.5，默认 0.85 偏舒缓
  utterance.pitch = 1.1;     // 略微偏高，更温柔
  utterance.volume = 0.8;
  window.speechSynthesis.speak(utterance);
}
```
